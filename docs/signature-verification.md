# Signature Verification

Every release of `provider-anthropic` is signed with [cosign](https://github.com/sigstore/cosign)
using keyless signing: `.github/workflows/publish-pkg.yaml`'s `sign-artifacts` job requests a
GitHub Actions OIDC token, exchanges it with Fulcio for a short-lived certificate bound to that
exact workflow, and publishes the signature and certificate to the public Rekor transparency log.
There is no private key to manage or leak — trust is anchored to the GitHub Actions identity of
the workflow that produced the release.

Both the primary image and its Upbound mirror are signed:

- `ghcr.io/jonasz-lasut/provider-anthropic:<version>`
- `xpkg.upbound.io/jonasz-lasut/provider-anthropic:<version>`

## Attestations

Alongside the signature, each image carries two attestations, both discoverable through the same
registry/Rekor mechanism cosign uses (`cosign verify-attestation`):

- **SBOM** (`spdxjson`) — generated with [syft](https://github.com/anchore/syft) from the
  published image and attached with `cosign attest`.
- **SLSA build provenance** (`slsaprovenance1`, predicate type
  `https://slsa.dev/provenance/v1`) — generated by
  [`actions/attest`](https://github.com/actions/attest), which auto-derives the provenance
  predicate (source repo, commit, workflow, run) from the GitHub Actions run itself rather than
  it being hand-assembled, and pushes it to the registry via `push-to-registry: true`.

## Verifying manually with the cosign CLI

To check a specific image yourself without going through Crossplane, use the
[cosign CLI](https://github.com/sigstore/cosign) with the same keyless identity Crossplane
verifies against — the workflow file, ref, issuer, so that a signature from *any* GitHub Actions
run doesn't pass:

```console
cosign verify \
  --certificate-identity https://github.com/jonasz-lasut/provider-anthropic/.github/workflows/publish-pkg.yaml@refs/heads/main \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  ghcr.io/jonasz-lasut/provider-anthropic:<version>
```

Verify each attestation the same way, adding `--type`:

```console
cosign verify-attestation \
  --certificate-identity https://github.com/jonasz-lasut/provider-anthropic/.github/workflows/publish-pkg.yaml@refs/heads/main \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  --type spdxjson \
  ghcr.io/jonasz-lasut/provider-anthropic:<version>

cosign verify-attestation \
  --certificate-identity https://github.com/jonasz-lasut/provider-anthropic/.github/workflows/publish-pkg.yaml@refs/heads/main \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  --type slsaprovenance1 \
  ghcr.io/jonasz-lasut/provider-anthropic:<version>
```

Swap in `xpkg.upbound.io/jonasz-lasut/provider-anthropic:<version>` to check the Upbound mirror
instead — same identity, since both images are signed and attested by the same workflow.

On success, `cosign verify`/`cosign verify-attestation` print the matching Rekor entries and exit
`0`; on failure they exit non-zero with the reason (no matching signature, identity mismatch,
missing attestation, …), which makes them easy to drop straight into a script or CI gate.
`verify-attestation` prints the full in-toto envelope — pipe through
`jq -r '.payload | @base64d | fromjson | .predicate'` to inspect just the predicate (the SPDX
document or SLSA provenance statement) rather than the signed wrapper.

## Configuring Crossplane to verify signatures and attestations

Crossplane can enforce both before installing a package via the
[`ImageConfig`](https://docs.crossplane.io/latest/packages/image-configs/) API. Attestations are
listed on the same authority used for the signature check — once that authority's signature is
verified, each listed `predicateType` must also have a validly-signed attestation present:

```yaml
apiVersion: pkg.crossplane.io/v1beta1
kind: ImageConfig
metadata:
  name: provider-anthropic-verification
spec:
  matchImages:
    - type: Prefix
      prefix: ghcr.io/jonasz-lasut/provider-anthropic
  verification:
    provider: Cosign
    cosign:
      authorities:
        - name: provider-anthropic release workflow
          keyless:
            identities:
              - issuer: https://token.actions.githubusercontent.com
                subject: https://github.com/jonasz-lasut/provider-anthropic/.github/workflows/publish-pkg.yaml@refs/heads/main
          attestations:
            - name: sbom
              predicateType: spdxjson
            - name: provenance
              predicateType: slsaprovenance1
```

Drop either entry from `attestations` if you only care about one of the two checks. Crossplane
currently only checks that a validly-signed attestation of the given predicate type exists — it
doesn't evaluate the attestation's contents against a policy.

To also verify the Upbound mirror, add a second `matchImages` entry (or a separate `ImageConfig`)
with `prefix: xpkg.upbound.io/jonasz-lasut/provider-anthropic` — the `verification` block is
identical, since both images are signed and attested by the same workflow identity.

Once applied, Crossplane reports the verification result as a `Verified` condition on the
resulting `ProviderRevision`, and refuses to install the package if verification fails.

Where possible, pin `ProviderRevision`/package references to a digest rather than a mutable tag —
verification is tied to the image digest at sign time, so pinning avoids any window between
signing and a tag being repointed.
